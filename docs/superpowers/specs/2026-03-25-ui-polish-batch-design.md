# UI Polish Batch — Design Spec

5 изменений UI, сгруппированных в 3 блока по затронутым компонентам.

## Блок 1: Главная страница — CategoryCard

### Изменение 1: Доход по категории переключается с месяца на год

**Текущее поведение:** `CategoryCard` всегда показывает `incomePerMonth` независимо от положения переключателя МЕС/ГОД.

**Целевое поведение:** при переключении на ГОД карточки категорий показывают годовой доход, как и главная сумма в `HeroIncome`.

**Реализация:**
- `MainPage` передаёт в `CategoryCard` уже вычисленный `income` (monthly или annual в зависимости от `mode`), аналогично тому как передаёт `income` в `HeroIncome`.
- Проп `incomePerMonth` переименовывается в `income`.
- Данные: `CategoryStats` уже содержит `totalIncomePerMonth` и `totalIncomePerYear`.

**Файлы:** `category-card.tsx`, `main-page.tsx`.

### Изменение 2: Процент доходности вместо доли в портфеле

**Текущее поведение:** под суммой дохода в карточке категории отображается `portfolioSharePercent` — доля категории в стоимости портфеля.

**Целевое поведение:** вместо доли показывается `yieldPercent` — годовая доходность категории.

**Реализация:**
- В `CategoryCard` проп `portfolioSharePercent` заменяется на `yieldPercent`.
- Данные: `CategoryStats` уже содержит `yieldPercent`.

**Файлы:** `category-card.tsx`, `main-page.tsx`.

## Блок 2: Страница категории — AssetRow

### Изменение 3: Выравнивание сумм дохода

**Текущее поведение:** бейджик источника («факт» / «ручной») стоит inline справа от суммы без фиксированной ширины. Разная длина текста бейджиков смещает суммы по горизонтали.

**Целевое поведение:** суммы выровнены по вертикали независимо от длины бейджика.

**Реализация (вариант A — фиксированная ширина бейджика):**
- Бейджик получает `min-w-[52px] text-center` — достаточно для «ручной» (6 символов моноширинного шрифта + padding).
- Порядок сохраняется: сумма → бейджик.

**Файлы:** `asset-row.tsx`.

### Изменение 4: Доходность актива в процентах

**Текущее поведение:** под строкой дохода показываются только количество и стоимость.

**Целевое поведение:** под строкой дохода и бейджика появляется `X.X% годовых`.

**Реализация:**
- Вычисление в компоненте: `calcYieldPercent(annualIncome * totalQuantity, value)`, где `value = currentPrice * totalQuantity`.
- Отображение: строка `font-mono text-caption text-muted` под flex-контейнером с суммой и бейджиком. Показывается только при `value > 0`.
- Функция `calcYieldPercent` уже импортируется в `income-calculator.ts`.

**Файлы:** `asset-row.tsx`.

## Блок 3: Меню + Настройки

### Изменение 5: Перенос бэкапа в настройки

**Текущее поведение:** в меню два пункта в секции «Прочее»: «Настройки» (`/settings`) и «Экспорт / Бэкап» (`/backup`). Функциональность разделена на две страницы.

**Целевое поведение:** единая страница «Настройки» содержит всё. Пункт «Экспорт / Бэкап» удалён из меню.

**Реализация:**
- В `settings-page.tsx` добавляются секции из `backup-page.tsx`:
  - «Экспорт» (кнопка скачать JSON)
  - «Восстановление» (кнопка загрузить JSON)
  - Статус/ошибка отображаются под кнопками
- Порядок секций на странице: Период по умолчанию → Экспорт → Восстановление → Опасная зона (удаление данных).
- Из `drawer-menu.tsx` удаляется пункт `{ label: 'Экспорт / Бэкап', path: '/backup', icon: Save }`.
- Маршрут `/backup` удаляется из роутера.
- Файл `backup-page.tsx` удаляется.

**Файлы:** `settings-page.tsx`, `drawer-menu.tsx`, роутер, `backup-page.tsx` (удаление).

## Не затрагивается

- `usePortfolioStats` — все необходимые данные (`yieldPercent`, `totalIncomePerYear`, `totalIncomePerMonth`) уже вычисляются.
- `income-calculator.ts` — `calcYieldPercent` уже экспортируется.
- Типы в `types.ts` — `CategoryStats` уже содержит все нужные поля.
- `HeroIncome` — без изменений.
